# 区块链 Indexer 设计

区块链节点更像原始数据源，不像业务数据库。业务系统需要 Indexer 持续读取区块，把链上交易转换成自己的表结构。

同步流程：

1. 读取链配置和当前模式。
2. mock 模式生成模拟交易，便于本地运行和面试演示。
3. rpc 模式通过 Web3j 调用测试网 RPC。
4. 读取区块中的交易或日志。
5. 解析原生币转账和 ERC20 Transfer。
6. 调用风控引擎打标。
7. 使用 `txHash + logIndex` 幂等保存。

`blockNumber` 用来定位区块，后续可以扩展 checkpoint 表，记录每条链已经同步到哪个高度。这样服务重启后可以从上次位置继续同步。

本 MVP 的 rpc 模式实现了区块交易读取，ERC20 日志解析器已经独立封装，后续可接入 `eth_getLogs` 按 Transfer topic 过滤日志。
