# 风控引擎设计

链上交易风控的目标不是替代合规系统，而是在交易进入业务系统时尽早打上风险标签，方便后续查询、告警和人工复核。

当前规则：

- 单笔金额大于 10000：`MEDIUM`。
- 单笔金额大于 100000：`HIGH`。
- `toAddress` 命中黑名单：`HIGH`。
- 短时间同一 `fromAddress` 高频转账：`MEDIUM`。
- `gasUsed` 异常高：`MEDIUM`。
- 交易失败：`LOW`。

规则结果由 `riskLevel` 和 `riskReason` 保存到交易表。这样查询交易时不需要重复计算，也方便解释接口直接生成自然语言说明。

这个项目和交易所后端、钱包后端的关系在于：交易所和钱包都需要地址画像、充值到账识别、提现风险控制、异常交易监控和链上数据归档。ChainLens 是这些能力的简化版。
